Reflection

In this course, I developed an advising assistance program for ABC University that loads course data from a file, stores it in a data structure, and allows users to print a sorted list of courses or search for a specific course and its prerequisites.

To solve this problem, I analyzed different data structures including a vector, hash table, and binary search tree. Understanding these structures was important because each one has different runtime and memory tradeoffs. By evaluating Big-O complexity, I was able to determine which structure best met the program’s needs.

One challenge I faced was handling file input correctly, especially when file paths contained spaces. I resolved this by switching from cin >> to getline() to properly read full paths. Debugging these issues strengthened my problem-solving skills and attention to detail.

This project improved how I approach software design. I learned to plan first using pseudocode and runtime analysis before coding. It also reinforced the importance of writing clear, modular, and maintainable programs.
